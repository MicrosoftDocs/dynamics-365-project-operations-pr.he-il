# <a name="link-actuals-to-original-records"></a>קישור נתונים בפועל לרשומות המקוריות

_**חל על:** Project Operations לתרחישים מבוססי משאבים/לא מלאי, פריסה קלה - עסקה בחשבונית פרופורמה_


ב- Dynamics 365 Project Operations, *עסקה* היא מושג מופשט שאינו מיוצג על-ידי ישות כלשהי. עם זאת, שדות ותהליכים נפוצים מסוימים בישויות נועדו להשתמש במושג של יחידות עסקיות. הישויות הבאות משתמשות ברעיון זה:

- פרטים בשורת הצעת מחיר
- פרטי סעיף חוזה
- שורות הערכה
- שורות יומן
- נתונים בפועל

מבין ישויות אלה, **פרטי שורת הצעת מחיר**, **פרטי סעיף חוזה** ו **‏‫שורות הערכה‬** ממופים לשלב ההערכה במחזור החיים של הפרוייקט. **שורות יומן** ו **ישויות הנתונים בפועל** ממופות לשלב הביצוע במחזור החיים של הפרוייקט.

Project Operations מתייחס לרשומות בחמש ישויות אלה כעסקאות. ההבחנה היחידה היא שרשומות בישויות הממופות לשלב ההערכה נחשבות לתחזיות פיננסיות, בעוד שהרשומות בישויות הממופות לשלב הביצוע נחשבות לעובדות פיננסיות שכבר התרחשו.

## <a name="concepts-that-are-unique-to-business-transactions"></a>מושגים הייחודיים לעסקאות
המושגים הבאים הם ייחודיים למושג של עסקאות:

- סוג עסקה
- מחלקת עסקה
- מקור עסקה
- חיבור עסקה

### <a name="transaction-type"></a>סוג עסקה

סוג עסקה מייצג את התזמון ואת ההקשר של ההשפעה הפיננסית על פרוייקט. הוא מיוצג באמצעות קבוצת אפשרויות בעלת הערכים הנתמכים הבאים ב- Project Operations:

  - עלות
  - חוזה הפרוייקט
  - מכירות שלא חויבו
  - מכירות שחויבו
  - מכירות בין-ארגוניות
  - עלות יחידת הקצאת משאבים

### <a name="transaction-class"></a>מחלקת עסקאות

מחלקת עסקה מייצגת את הסוגים השונים של עלויות המושקעות בפרוייקטים. הוא מיוצג באמצעות קבוצת אפשרויות בעלת הערכים הנתמכים הבאים ב- Project Operations:

  - זמן
  - הוצאה
  - חומר
  - תשלום
  - אבן דרך
  - סה"כ מס

הערך **אבן דרך** משמש בדרך כלל את הלוגיקה העסקית עבור חיוב בעל מחיר קבוע ב- Project Operations.

### <a name="transaction-origin"></a>מקור עסקה

**מקור עסקה** הוא ישות המאחסנת את המקור של כל עסקה. כאשר פרויקט יוצא לדרך, כל עסקה תיצור עסקה נוספת אשר בתורה תיצור עוד עסקה וכן הלאה. ישות מקור עסקה נועדה לאחסן נתונים לגבי מקור של כל עסקה כדי לסייע בדיווח וביכולת מעקב. 

### <a name="transaction-connection"></a>חיבור עסקה

**חיבור עסקה** הוא ישות המאחסנת את הקשר בין שתי עסקאות דומות, כגון עלות ונתונים בפועל של מכירות קשורות, או ביטולי עסקאות המופעלים על-ידי פעילויות חיוב, כגון אישור חשבונית או תיקוני חשבונית.

יחד, הישויות **מקור עסקה** ו **מקור עסקה** עוזרות לך לעקוב אחר קשרים בין עסקאות ופעולות שגרמו ליצירה של עסקה ספציפית.

### <a name="example-how-transaction-origin-works-with-transaction-connection"></a>דוגמה: כיצד מקור עסקה פועל עם חיבור עסקה

הדוגמה הבאה מציגה את העיבוד האופייני של ערכי זמן במחזור חיים של פרוייקט Project Operations.

> ![ערכי זמן העיבוד במחזור חיים של Project Service.](media/basic-guide-17.png)
 
1. הגשת ערך זמן יוצרת שתי שורות יומן: שורה אחת עבור עלות ושורה אחת עבור מכירות שלא חויבו.
2. אישור סופי של ערך הזמן יוצר שני ערכים של נתונים בפועל: אחד עבור נתונים בפועל של עלות ואחד עבור נתונים בפועל של מכירות שלא חויבו.
3. כאשר נוצרת חשבונית פרוייקט חדשה, עסקה בשורת החשבונית נוצרת על-ידי שימוש בנתונים בפועל של מכירות שלא חויבו. 
4. כאשר החשבונית מאושרת, נוצרים שני נתונים בפועל חדשים: נתונים בפועל של ביטול מכירות שלא חויבו ונתונים בפועל של מכירות שחויבו.

כל אחד מהאירועים הללו יוצר רשומה בישויות **מקור עסקה** ו **חיבור עסקה**. רשומות חדשות אלה מסייעות בבניית היסטוריה של קשרים בין הרשומות שנוצרות בערך זמן, בשורת יומן, בנתונים בפועל ובפרטי שורת חשבונית.
