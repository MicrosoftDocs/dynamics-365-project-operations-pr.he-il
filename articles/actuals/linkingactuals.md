# <a name="transaction-origins---link-actuals-to-their-source"></a>מקורות עסקאות - קשר את הנתונים בפועל למקור שלהם

_**חל על:** Project Operations לתרחישים מבוססי משאבים/לא מלאי, פריסה קלה - עסקה בחשבונית פרופורמה_

רשומות מקור העסקה נוצרות כדי לקשר נתונים בפועל למקור שלהן, כגון ערכי זמן, ערכי הוצאות, יומני שימוש בחומרים וחשבוניות פרויקטים.

הדוגמה הבאה מציגה את העיבוד האופייני של ערכי זמן במחזור חיים של פרוייקט Project Operations.

> ![עיבוד ערכי זמן ב -Project Operations.](media/basic-guide-17.png)
 
1. הגשת ערך זמן גורמת ליצירת שתי שורות יומן: אחת עבור עלות ואחת עבור מכירות שלא חויבו.
2. אישור סופי של ערך הזמן גורם ליצירת שני נתונים בפועל: אחד עבור עלות ואחד עבור מכירות שלא חויבו.
3. כאשר המשתמש יוצר חשבונית פרוייקט, עסקת שורה בחשבונית נוצרת על-ידי שימוש בנתונים מהמכירות בפועל שלא חויבו.
4. כאשר החשבונית מאושרת, שני הנתונים בפועל החדשים נוצרים: ביטול מכירות שלא חויבו ומכירות בפועל שחויבו.

כל אירוע בזרימת העבודה של עיבוד מפעיל את יצירת הרשומות בישות מקור העסקה כדי לסייע בבניית מעקב אחר קשרים בין רשומות אלה שנוצרו בפרטי ערך זמן, שורת יומן, נתון בפועל ושורה בחשבונית.
