# <a name="actuals"></a>נתונים בפועל 

_**חל על:** Project Operations לתרחישים מבוססי משאבים/לא מלאי, פריסה קלה - עסקה בחשבונית פרופורמה_

‏‫נתונים בפועל‬ מייצגים את ההתקדמות הכספית והתקדמות לוח הזמנים שנבדקה ואושרה בפרויקט. הם נוצרים כתוצאה מאישור זמן, הוצאות, ערכי שימוש בחומרים וערכים ביומן וחשבוניות.

## <a name="journal-lines-and-time-submission"></a>שורות יומן והגשת זמן

לקבלת מידע נוסף אודות ערך זמן, ראה [מבט כולל על ערך זמן](../time/time-entry-overview.md).

### <a name="time-and-materials"></a>זמן וחומרים

כאשר ערך זמן שמוגש מקושר לפרויקט הממופה לסעיף חוזה של זמן וחומרים, המערכת יוצרת שתי שורות יומן, אחת עובר עלות ואחת עבור מכירות שלא חויבו.

### <a name="fixed-price"></a>מחיר קבוע

כאשר ערך זמן שמוגש מקושר לפרוייקט הממופה לסעיף חוזה במחיר קבוע, המערכת יוצרת שורת יומן אחת עבור עלות.

### <a name="default-pricing"></a>תמחור ברירת מחדל

הלוגיקה ליצירת מחירי ברירת מחדל נמצאת בשורת היומן. ערכי השדות מערך הזמן מועתקים לשורת היומן. ערכים אלה כוללים את תאריך העסקה, את סעיף החוזה שאליו ממופה הפרוייקט ואת תוצאת המטבע במחירון המתאים.

שדות המשפיעים על תמחור ברירת המחדל, כגון **תפקיד** ו **‏‫יחידת הקצאת משאבים‬**, משמשים לקביעת המחיר המתאים בשורת היומן. ניתן להוסיף שדה מותאם אישית בערך הזמן. אם ברצונך שערך השדה יופץ לנתונים בפועל, צור את השדה בטבלאות **נתונים בפועל** ו **שורת יומן**. השתמש בקוד מותאם אישית כדי להפיץ את ערך השדה שנבחר מ'‏‫ערך זמן‬' ל'נתונים בפועל' דרך שורת היומן באמצעות מקורות העסקה. למידע נוסף על מקורות העסקה וחיבורים, ראה [קישור נתונים בפועל לרשומות מקוריות](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection).

## <a name="journal-lines-and-basic-expense-submission"></a>שורות יומן והגשת הוצאות בסיסיות

לקבלת מידע נוסף אודות ערכי הוצאה, ראה [מבט כולל על ערכי הוצאה](../expense/expense-overview.md).

### <a name="time-and-materials"></a>זמן וחומרים

כאשר ערך הוצאה בסיסית שמוגש מקושר לפרויקט הממופה לסעיף חוזה של זמן וחומרים, המערכת יוצרת שתי שורות יומן, אחת עבור עלות ואחת עבור מכירות שלא חויבו.

### <a name="fixed-price"></a>מחיר קבוע

כאשר ערך הוצאה בסיסית שנשלח מקושר לפרוייקט הממופה לסעיף חוזה של מחיר קבוע, המערכת יוצרת שורת יומן אחת עבור עלות.

### <a name="default-pricing"></a>תמחור ברירת מחדל

הלוגיקה להזנת מחירי ברירת מחדל להוצאות מבוססת על קטגוריית ההוצאות. תאריך העסקה, סעיף החוזה שאליו ממופה הפרוייקט והמטבע משמשים כולם כדי לקבוע את המחירון המתאים. שדות המשפיעים על תמחור ברירת המחדל, כגון **‏‫קטגוריית עסקה‬** ו **‏‫יחידה‬**, משמשים לקביעת המחיר המתאים בשורת היומן. עם זאת, זה עובד רק כאשר שיטת התמחור במחירון היא **מחיר ליחידה**. אם שיטת התמחור היא **‏‫לפי עלות‬** או **ייקור מעל לעלות**, המחיר שהוזן בעת ערך ההוצאה משמש עבור עלות, והמחיר בשורת יומן המכירות מחושב לפי שיטת התמחור. 

באפשרותך להוסיף שדה מותאם אישית בערך ההוצאה. אם ברצונך שערך השדה יופץ לנתונים בפועל, צור את השדה בטבלאות **נתונים בפועל** ו **שורת יומן**. השתמש בקוד מותאם אישית כדי להפיץ את ערך השדה שנבחר מ'‏‫ערך זמן‬' ל'נתונים בפועל' דרך שורת היומן באמצעות מקורות העסקה. למידע נוסף על מקורות העסקה וחיבורים, ראה [קישור נתונים בפועל לרשומות מקוריות](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection).

## <a name="journal-lines-and-material-usage-log-submission"></a>הגשת יומן של שורות יומן ושימוש בחומרים

לקבלת מידע נוסף אודות הזנת הוצאה, ראה [יומן שימוש בחומרים](../material/material-usage-log.md).

### <a name="time-and-materials"></a>זמן וחומרים

כאשר ערך ביומן השימוש בחומרים שנשלח מקושר לפרויקט הממופה לשורת חוזה של זמן וחומרים, המערכת יוצרת שתי שורות יומן, אחת עבור עלות ואחת עבור מכירות שלא חויבו.

### <a name="fixed-price"></a>מחיר קבוע

כאשר ערך ביומן השימוש בחומרים שנשלח מקושר לפרוייקט הממופה לסעיף חוזה של מחיר קבוע, המערכת יוצרת שורת יומן אחת עבור עלות.

### <a name="default-pricing"></a>תמחור ברירת מחדל

ההיגיון להזנת מחירי ברירת מחדל עבור חומר מבוסס על שילוב המוצר והיחידה. תאריך העסקה, סעיף החוזה שאליו ממופה הפרוייקט והמטבע משמשים כולם כדי לקבוע את המחירון המתאים. שדות המשפיעים על תמחור ברירת המחדל, כגון **מזהה מוצר** ו **‏‫יחידה‬**, משמשים לקביעת המחיר המתאים בשורת היומן. עם זאת, זה עובד רק עבור מוצרי קטלוג. עבור מוצרים שאינם מופיעים ברשימה, המחיר שהוזן בעת יצירת ערך ביומן השימוש בחומרים משמש עבור עלות ומחיר מכירה בשורות היומן. 

באפשרותך להוסיף שדה מותאם אישית בערך **‏‫יומן רישום של שימוש בחומרים‬**. אם ברצונך שערך השדה יופץ לנתונים בפועל, צור את השדה בטבלאות **נתונים בפועל** ו **שורת יומן**. השתמש בקוד מותאם אישית כדי להפיץ את ערך השדה שנבחר מ'‏‫ערך זמן‬' ל'נתונים בפועל' דרך שורת היומן באמצעות מקורות העסקה. למידע נוסף על מקורות העסקה וחיבורים, ראה [קישור נתונים בפועל לרשומות מקוריות](linkingactuals.md#example-how-transaction-origin-works-with-transaction-connection).

## <a name="use-entry-journals-to-record-costs"></a>שימוש ביומני ערכים לתיעוד עלויות

יומני ערכים מאפשרים לך לתעד עלות או הכנסה במחלקות החומר, התשלום, הזמן, ההוצאה או עסקאות מס. ניתן להשתמש ביומנים למטרות הבאות:

- נתונים בפועל נוספים על עסקאות ממערכת אחרת ל- Microsoft Dynamics 365 Project Operations.
- רישום עלויות שהתרחשו במערכת אחרת. עלויות אלה יכולות לכלול עלויות רכש או קבלנות משנה.

> [!IMPORTANT]
> היישום אינו מאמת את סוג שורת היומן או את התמחור הקשור שמזינים בשורת היומן. לכן, כדאי שרק משתמש שמודע לחלוטין להשפעה החשבונאית שיש נתונים בפועל על הפרויקט, ישתמש ביומני ערך ליצירת נתונים בפועל. בגלל ההשפעה של סוג יומן זה, עליך לבחור בקפידה למי יש גישה ליצירת יומני ערך.
